# Моя фильмотека

Личный каталог просмотренных фильмов, сериалов и аниме. Статичный сайт (GitHub Pages) + облачная база Supabase — чистый HTML/CSS/JS, без сборки.

## Структура

- `index.html` — главная: вкладки «Главная / Топ 100 / Новинки», фильтры, карточки-строки
- `film.html?id=...` — страница произведения: детали, серия фильмов, админ-панель (для владельца)
- `admin.html` — инструмент добавления (ссылки в меню нет)
- `js/db.js` — работа с базой Supabase (каталог, добавление/обновление/удаление, вход)
- `js/app.js`, `js/film.js`, `js/admin.js`, `css/style.css`
- `supabase-setup.sql` — схема базы и стартовые данные (выполняется один раз в SQL Editor)
- `data.json` — резервная копия каталога (сайтом больше не используется)

## Как всё устроено

- **Данные** живут в Supabase (таблица `films`). Чтение открыто всем — сайт публичный.
  Запись защищена RLS-политиками: добавлять/менять/удалять может только вошедший администратор.
- **URL и publishable-ключ** Supabase в `js/db.js` публичны по дизайну — безопасность
  обеспечивают политики базы, а не секретность ключа.
- **Добавление**: `admin.html` → вход по email → поиск в TMDb (русские данные) →
  добор рейтинга IMDb из OMDb → «Добавить» → запись появляется на сайте мгновенно.
- **Серии фильмов**: коллекция TMDb «запекается» в запись при добавлении;
  на странице фильма показывается ветка всех частей с пометкой непросмотренных.
- Ключи TMDb/OMDb хранятся в localStorage браузера администратора.

## Хостинг

GitHub Pages, деплой через GitHub Actions (`.github/workflows/deploy.yml`) при пуше в `main`.
Деплой нужен только для изменений кода — данные обновляются без деплоя.
